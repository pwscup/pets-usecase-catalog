# プロンプト: 既存事例を追加文献で補完・改善する

既存の事例を、追加の文献情報でより充実させるためのプロンプトです。「調査中」の項目だけでなく、すべての項目がより正確・詳細になるよう改善します。

---

## 使い方

1. 下記のプロンプトをコピーする
2. 補完したい事例の `case.json` の内容を貼り付ける
3. 追加の文献URL・PDF・テキストを貼り付ける
4. AIに送信する
5. 出力された更新版JSONを事例編集画面の「AIの出力をインポート」に貼り付ける

> **Tip**: 事例編集画面にある「AI で入力を補助する」パネルには、現在の事例JSONが埋め込まれたプロンプトが自動生成されます。

---

## プロンプト本文

````
# 既存事例の補完・改善

## 入力情報

以下に追加文献のURL、PDF、またはテキスト情報を貼り付けてください：

【ここに追加文献のURL、PDF、またはテキストを貼り付け】

---

上記の情報をもとに、以下の既存事例JSONを補完・改善してください。

## 既存の事例JSON
```json
【ここに既存の case.json の内容を貼り付け】
```

## 作業ルール

1. **更新対象**: すべての項目が改善の対象です。「調査中」の項目はもちろん、既存の記載がある項目でも、追加文献からより正確・詳細・具体的な情報が得られる場合は積極的に更新してください
2. **改善の例**:
   - 「調査中」→ 具体的な内容に更新
   - 抽象的な記述 → より具体的な記述（数値、手法名、組織名など）
   - 不完全な情報 → 文献で確認できた情報で補完
   - summary や value_proposition の表現をより分かりやすく改善
3. **sources の追加**: 追加文献を sources 配列に追記する（既存のsourceは削除しない）
4. **推測しない**: 文献に明記されていない情報は既存の記載を維持する（「調査中」はそのまま残す）

## data_flow 図の改善

既存の図のノードについても、追加文献の情報で label をより具体的にできる場合は更新してください。
例:
- 「合成データを生成」→「CTGANで合成データを生成」（手法名が判明した場合）
- 「成果を達成」→「検出精度95%を達成」（数値が判明した場合）
- ノードの追加・分割も、情報が充実した場合は積極的に行ってください

## 出力形式

更新後の完全なJSONを出力してください。
JSONの後に、変更点のサマリーを箇条書きで示してください：
- 更新した項目と内容
- まだ「調査中」のまま残っている項目
- 新たに判明した情報の出典箇所
````
